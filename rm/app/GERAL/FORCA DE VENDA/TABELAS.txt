REMOTE SALE

Baixa o aplicativo, digita o cnpj da empresa e pede homologacao.

O aparelho aparece no painel e fica aguardando ativação. Dá entrada no cadastro com o numero do id do aparelho.

Administrador ativa ou desativa o aparelho no painel

Se ativado:
O vendedor pode fazer o login


ADMINISTRATIVO

Cadastra o cliente no portal usando apenas o CNPJ
Cria token unico de acesso
Define status como aguardando

AWS:
- Cria um bucket com o cnpj do cliente
- Cria 5 sub-buckets
gadgets / vendedores / clientes / produtos / pedidos


CONCILIAÇÃO BD
- Cadastra os nomes das colunas do bd de origem e define com qual coluna padrão ela é compativel.

- faz a troca dos nomes das colunas

- insere as tabelas no bucket correspondente

--

Sempre que houver inserção de dados nos buckets

Define atualização no cadastro do cliente.

O PouchDB visualiza a alteração e chama os arquivos do S3 e faz alteração no banco de dados local.
